# Getting Started Guide

This guide walks you through setting up and running this template from scratch after cloning.

---

## Prerequisites
Before starting, ensure you have the following installed:
- [Node.js](https://nodejs.org/) (v18 or higher recommended)
- [pnpm](https://pnpm.io/) (the preferred package manager for this project)

---

## Step 1: Install Dependencies
Clone the repository, navigate to the project root, and install the required dependencies:
```bash
pnpm install
```

---

## Step 2: Set Up Convex (Backend)
This template uses [Convex](https://convex.dev) as its backend.
1. Run the development command to initialize and link your Convex project:
   ```bash
   pnpm run convex:gen
   ```
   *Or start the dev server directly (this will prompt you to log into Convex and create a new project if you haven't already):*
   ```bash
   npx convex dev
   ```
2. This will automatically generate your local configuration and create a `.env.local` file containing your Convex deployment keys and URLs (e.g., `CONVEX_DEPLOYMENT`, `NEXT_PUBLIC_CONVEX_URL`, and `NEXT_PUBLIC_CONVEX_SITE_URL`).

---

## Step 3: Set Up Firebase (Authentication & Client)
This template uses Firebase for user authentication. You will need to create a Firebase project and obtain the client configuration environment variables.

### 1. Create a Firebase Project
1. Go to the [Firebase Console](https://console.firebase.google.com/).
2. Click **Add project** and follow the prompts to create a new project.
3. Once created, register a new **Web App** in the project dashboard (click the Web icon `</>`).
4. Copy the `firebaseConfig` object values from the setup page. They look like this:
   ```javascript
   const firebaseConfig = {
     apiKey: "YOUR_API_KEY",
     authDomain: "YOUR_PROJECT_ID.firebaseapp.com",
     projectId: "YOUR_PROJECT_ID",
     storageBucket: "YOUR_PROJECT_ID.firebasestorage.app",
     messagingSenderId: "YOUR_SENDER_ID",
     appId: "YOUR_APP_ID"
   };
   ```

### 2. Configure Environment Variables
Create a file named `.env.local` at the root of the project (if it wasn't already generated by Convex, or edit the existing one) and add your Firebase credentials:

```env
# Firebase Client Configuration
NEXT_PUBLIC_FIREBASE_API_KEY="YOUR_API_KEY"
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN="YOUR_PROJECT_ID.firebaseapp.com"
NEXT_PUBLIC_FIREBASE_PROJECT_ID="YOUR_PROJECT_ID"
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET="YOUR_PROJECT_ID.firebasestorage.app"
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID="YOUR_SENDER_ID"
NEXT_PUBLIC_FIREBASE_APP_ID="YOUR_APP_ID"
```

---

## Step 4: Enable Authentication Providers in Firebase
To allow users to log in, you must enable authentication providers in the Firebase Console.

### 1. Enable Email/Password or Social Logins
1. In the Firebase Console left-sidebar, navigate to **Build** > **Authentication**.
2. Click **Get Started** if this is your first time opening it.
3. Switch to the **Sign-in method** tab.
4. Click **Add new provider** under the **Additional providers** or **Native providers** list.

### 2. Setting Up Google Social Login
1. Select **Google** from the provider options.
2. Toggle the **Enable** switch.
3. Select a project support email from the dropdown menu.
4. Click **Save**.
   > [!NOTE]
   > For local development, Google Sign-in works out of the box because Firebase automatically whitelists `localhost` by default.

---

## Step 5: Production Configuration (Authorized Domains)
When deploying your app to production (e.g., on Vercel or Netlify), you must configure authorization domains in Firebase, otherwise authentication requests from your custom domain will be blocked.

### 1. Authorize Your Production Domain in Firebase
1. Go to the **Firebase Console** > **Authentication** > **Settings** tab.
2. Select **Authorized domains** from the left-hand menu.
3. Click **Add domain**.
4. Enter your production domain name (e.g., `myapp.vercel.app` or `www.mycustomdomain.com`).
5. Click **Add**.

### 2. Configure Google Cloud Console (For Social Logins in Production)
If using Google Sign-in, you must also whitelist your production domain in the Google Cloud Console:
1. Go to the [Google Cloud Console Credentials Page](https://console.cloud.google.com/apis/credentials).
2. Select your Firebase project from the top dropdown list.
3. Under **OAuth 2.0 Client IDs**, click edit (pencil icon) next to the **Web client (auto created by Google Service)** credential.
4. Under **Authorized JavaScript origins**, add your production URL (e.g., `https://myapp.vercel.app`).
5. Under **Authorized redirect URIs**, add the Firebase authorization redirect URI (e.g., `https://YOUR_PROJECT_ID.firebaseapp.com/__/auth/handler`).
6. Click **Save**.

---

## Step 6: Start the Development Server
Once both Convex and Firebase are configured, run the following command to start the Next.js development server:
```bash
pnpm run dev
```

Your app will be running at [http://localhost:3000](http://localhost:3000).
